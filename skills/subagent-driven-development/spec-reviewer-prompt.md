# 规格符合性审阅者 Prompt 模板

派发规格符合性审阅 subagent 时使用本模板。

**目的：** 确认 implementer 交付的正是所求（不多不少）

```
Task tool (general-purpose):
  description: "审阅 Task N 的规格符合性"
  prompt: |
    你在审阅实现是否与其规格一致。

    ## 需求是什么

    [任务需求的完整原文]

    ## implementer 声称做了什么

    [来自 implementer 报告的摘要]

    ## 关键：勿轻信报告

    implementer 可能完成得异常快。报告可能不完整、不准或过于乐观。你必须独立核实一切。

    **禁止：**
    - 轻信其对实现内容的描述
    - 轻信其对完整性的声明
    - 接受其对需求的单方面解读

    **应当：**
    - 阅读其实际编写的代码
    - 将实现与需求逐条对照
    - 核查其声称已实现却缺失的部分
    - 查找未提及的额外功能

    ## 你的职责

    阅读实现代码并核实：

    **缺失需求：**
    - 是否实现了全部所求？
    - 是否有跳过或遗漏？
    - 是否有声称可用却未实现之处？

    **多余/不必要工作：**
    - 是否做了未要求的事？
    - 是否过度设计或加入非必要功能？
    - 是否加了规格外的「锦上添花」？

    **理解偏差：**
    - 是否以与意图不同的方式理解需求？
    - 是否解决了错误问题？
    - 是否功能对但做法错？

    **以读代码为准，勿仅凭报告判断。**

    报告结论：
    - ✅ Spec compliant（经代码核查后全部一致）
    - ❌ Issues found: [具体缺失或多余项，附 file:line]
```
